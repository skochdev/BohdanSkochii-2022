you should store the state in the closest common parent of those siblings.

Handling errors ^QQ3Q8WLI

We must keep the error in our state. ^1RPIP8JU

Then, if we've caught an error, we can display something to the user.
 ^ZMZPrVTT

Normally, fetch() doesn't catch 404 errors, unless backend is able to response
with the status code 404.

Techincally, we should check whether response is successful with response.ok,
and if not we should return Propise.reject(new Error(`Your error text`)

404 will be caught on "response.ok", and message form new Error() will be passed
to the catch() method ^ZPRN5Phl

AbortController() TO STOP HTTP REQUEST ^Rb7AFDJx

To clean up HTTP requests (just like event listeners
or setTimeouts) you can use AbortController()
in componentWillUnmount(). For example, while HTTP request is on process
user might change the page, and the component will unmount, but 
async http request will still be running. Clean it up  ^H5SptSvc
